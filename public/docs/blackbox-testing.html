Test Plan

1. Introduction

This document defines the blackbox testing strategy for Dompet Patungan, a group expense-splitting and settlement management web application. Testing is conducted without knowledge of internal code structure; test cases are derived exclusively from functional requirements, user interface specifications, and business rules.

2. Test Objectives

  • Verify that all functional requirements are implemented correctly
  • Ensure proper access control and authorization for each user role
  • Validate input handling, error messages, and boundary conditions
  • Confirm that business rules (settlement calculation, notification delivery, group lifecycle) behave as specified
  • Identify defects before production release
